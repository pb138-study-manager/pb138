# Notes API (with Folders) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement CRUD API for personal notes and folders — same auth/ownership/soft-delete/audit pattern as Tasks and Events.

**Architecture:** Two route files (`folders.ts`, `notes.ts`) registered in `index.ts`. Schema updated to add `folders` table and two new columns to `notes`. DB migration applied before any tests. All tests use a real PostgreSQL database with JWT auth via `jose`.

**Tech Stack:** ElysiaJS, Drizzle ORM, Bun test, PostgreSQL, Supabase JWT (`jose`)

export const events = pgTable('events', {
  id: serial('id').primaryKey(),
  userId: integer('user_id')
    .notNull()
    .references(() => users.id),
  title: text('title').notNull(),
  description: text('description'),
  startDate: timestamp('start_date').notNull(),
  endDate: timestamp('end_date').notNull(),
  place: text('place'),

async function makeAuthHeader(): Promise<string> {
  const secret = new TextEncoder().encode(TEST_SECRET);

    .sign(secret);
