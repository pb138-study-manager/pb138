3. Postup inštalácie a konfigurácie

Krok 1 — Klonovanie repozitára

git clone https://github.com/pb138-study-manager/pb138.git
cd pb138

Krok 2 — Inštalácia závislostí

pnpm install

Krok 3 — Vytvorenie Supabase projektu

  1. Prihláste sa na supabase.com a vytvorte nový projekt.
  2. Po vytvorení projektu si poznačte tieto hodnoty z Project Settings → API:
    • Project URL (napr. https://abcdefgh.supabase.co)
    • anon / public key
    • service_role key (skrytý — kliknite na “Reveal”)
  3. Z Project Settings → JWT Settings si poznačte:
    • JWT Secret
  4. V Supabase dashboarde prejdite do Authentication → URL Configuration a nastavte:
    • Site URL: http://localhost:3000
    • Redirect URLs: http://localhost:3000/verify-email

Krok 4 — Konfigurácia premenných prostredia

Skopírujte vzorové .env súbory a vyplňte hodnoty:

cp apps/backend/.env.example apps/backend/.env
cp apps/frontend/.env.example apps/frontend/.env

apps/backend/.env:

DATABASE_URL=postgresql://postgres:heslo@localhost:5432/pb138
PORT=3001
NODE_ENV=development
SUPABASE_URL=https://vas-projekt.supabase.co
SUPABASE_JWT_SECRET=vas-jwt-secret
SUPABASE_SERVICE_ROLE_KEY=vas-service-role-key

apps/frontend/.env:

VITE_SUPABASE_URL=https://vas-projekt.supabase.co
VITE_SUPABASE_ANON_KEY=vas-anon-key

Krok 5 — Inicializácia databázy

cd apps/backend

# Vytvorte tabuľky (spustite migrácie)
bun run db:migrate

# Naplňte základné dáta (role, permissions, email šablóny)
bun run src/db/seed.ts

4. Postup spustenia aplikácie

Spustite backend a frontend v dvoch samostatných termináloch:

Terminál 1 — Backend:

cd apps/backend
bun run dev

Terminál 2 — Frontend:

pnpm --filter @pb138/frontend dev

Po spustení sú služby dostupné na:

Služba URL
Frontend (webová aplikácia) http://localhost:3000
Backend API http://localhost:3001

Prvé použitie — registrácia

  1. Otvorte aplikáciu na http://localhost:3000.
  2. Kliknite na Register a vytvorte účet.
  3. Skontrolujte email a kliknite na overovací odkaz (Supabase pošle email).
  4. Po overení sa prihláste — aplikácia vás presmeruje na Today stránku.
  5. Backend automaticky synchronizuje Supabase používateľa s lokálnou databázou pri prvom prihlásení.

5. Riešenie bežných problémov

Problém Príčina Riešenie
Backend sa nespustí Chybajúce .env premenné Skontrolujte apps/backend/.env — všetky Supabase hodnoty musia byť vyplnené
Registrácia nefunguje Nesprávna Supabase konfigurácia Overte Site URL a Redirect URL v Supabase dashboarde
Databáza: relation does not exist Migrácie neboli spustené Spustite bun run db:migrate v apps/backend
Port 3000 alebo 3001 obsadený Iný proces na porte Zmeňte port v apps/frontend/vite.config.ts resp. apps/backend/.env
