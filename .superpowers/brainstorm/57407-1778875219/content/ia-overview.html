Information Architecture

Section 1 of 5 — How the app is organized for each role. Does this structure feel right?

STUDENT Navigation
🏠 Today
AI-curated daily view · week strip · tasks for today · upcoming deadlines
✅ Tasks
Personal tasks · subtasks · AI priority scoring · teacher-assigned tasks visible here too
📚 Courses
Enrolled courses · progress per course · study materials from teacher · course tasks
📝 Notes
Personal notes · folders · link notes to courses · AI summarization
📅 Timeline
Events · deadlines from courses · AI-suggested study blocks
🤖 AI Copilot
Chat interface · weekly plan generation · workload warnings · note summaries
👤 Profile / Settings
Avatar · password · theme · notifications · language
TEACHER Navigation
🏫 My Classes
Courses you teach · class roster · today's sessions · pending evaluations count
📋 Assignments
Create & manage assignments · assign to course · set due dates · track submission status
📊 Evaluations
Grade submitted work · score + feedback · view per-student progress across course
📦 Study Materials
Upload/link materials to courses · visible to enrolled students · organize by topic
👥 Students
Enroll / remove students per course · search by name/email · view per-student task status
🤖 AI Insights
Class performance overview · students at risk · AI-drafted assignment descriptions
👤 Profile / Settings
Same as student profile section
Key principle: A teacher who is also a student (e.g. a PhD student) sees both nav sets — a role toggle at the top of the sidebar switches context. No separate login needed.
